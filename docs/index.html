Why choose our SDK

Everything a viral loop needs.
Nothing it doesn't.

01
Full funnel attribution
One call to handleDeepLink(intent) resolves generated → click → install → attributed. No SDK soup, no fragile Intent parsing, no guesswork.
02
Offline-first by design
Room-backed reads, WorkManager-backed writes with exponential backoff. generateInviteLink() falls back to a deterministic per-user code — the UI never blocks.
03
Drop-in Compose UI
ReferralBottomSheet is a single composable: invite CTA, native share handoff, and confetti included. Inherits your Material 3 theme automatically on day one.
04
Reactive & type-safe
observePointsBalance(): StateFlow<Int> drops straight into Compose. Typed errors — RateLimited, InsufficientPoints, AlreadyClaimed — mean precise UX, not a generic "something went wrong."
05
Remote config, zero redeploys
Change points_per_referral, the welcome bonus, or caps from the Campaign Manager. The SDK hot-reloads on its next getConfig() call — no app update required.
06
Anti-fraud, built in
Token-bucket rate limiting (5 attributions/min), 7-day replay nonces, self-referral blocking. The server is the single source of truth for every point balance.

Quickstart — Android / Kotlin

From zero to viral loop
in three steps.

Requirements: minSdk 23  ·  Kotlin 2.0+  ·  AGP 8.2+  ·  Java 17

1

Add the dependency

Register the SDK's Maven repository in your root settings.gradle.kts, then add the SDK to your app module.

// settings.gradle.kts
dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        maven("https://maven.referralvirality.io/releases")
    }
}
// app/build.gradle.kts
dependencies {
    implementation("io.referralvirality:referral-sdk:1.0.0")
}
Zero runtime permissions. The SDK merges INTERNET and ACCESS_NETWORK_STATE via manifest merging automatically.
2

Initialize the SDK

Call ReferralSdk.init() exactly once in Application.onCreate() — it's idempotent and safe. After sign-in, bind the user's stable ID.

class MyApp : Application() {
    override fun onCreate() {
        super.onCreate()
        ReferralSdk.init(
            context   = this,
            apiKey    = BuildConfig.REFERRAL_API_KEY,  // keep secrets out of source
            projectId = "proj_abc123"
        )
    }
}

// After your user authenticates
ReferralSdk.setUserId(authRepository.currentUserId)
Key Point: Use a stable, non-PII identifier — Firebase UID or your DB primary key. Never email or display name. It must stay constant for the same user across sessions.
3

Launch the referral flow

Drop the ready-made invite sheet into your Compose tree and handle the incoming deep link to close the attribution loop.

@Composable
fun GrowthCta() {
    var showInvite by remember { mutableStateOf(false) }

    // Live, reactive balance — straight from the SDK
    val points by ReferralSdk.observePointsBalance().collectAsState()

    Column {
        Text("You have $points pts")
        Button(onClick = { showInvite = true }) {
            Text("Invite friends")
        }
    }

    if (showInvite) {
        // Invite link, native share handoff, confetti — all included
        ReferralBottomSheet(onDismiss = { showInvite = false })
    }
}
// In your launcher Activity — closes the attribution loop
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    ReferralSdk.handleDeepLink(intent)        // cold start
}

override fun onNewIntent(intent: Intent) {
    super.onNewIntent(intent)
    ReferralSdk.handleDeepLink(intent)        // warm start
}
<!-- AndroidManifest.xml — intent filter on your launcher activity -->
<intent-filter>
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="referralsdk" android:host="invite" />
</intent-filter>
Tip: No device handy? Simulate a referral install via ADB:

adb shell am start -a android.intent.action.VIEW -d "referralsdk://invite?code=DEMO42" your.app.id
